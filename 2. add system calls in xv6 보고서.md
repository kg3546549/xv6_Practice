## 2. System Call에 대한 이해

---

### System Call이란?

System Call은 Application이 Kernal을 **제어**할 수 있는 인터페이스이다.

Application이 Process 내에서 Kernal에게 직접 접근하여 OS를 직접 제어한다.

User Process는 하드웨어를 직접 제어할 수 없기 때문에 Kernal에게 System Call을 통해 H/W를 제어한다.

하드웨어를 직접 제어할 수 없는 이유는, 하드웨어는 일반적으로 동시에 여러 작업을 할 수 없기 때문이다.

System Call을 호출하게 되면 커널 모드로 전환되어 커널에서만 접근이 가능한 데이터를 처리한 후, 결과값을 System Call을 한 프로세스에 Return한다.

> **System Call의 흐름** : 사용자 프로세스 → 시스템 콜 → 커널 처리 → 결과 반환
> 

이번 과제를 통해 System Call을 직접 만들어 볼 것이다.

## 3. getppid System Call 기능 분석

---

getppid (get Parent Process ID)는 이름 그대로 부모 프로세스의 PID를 return하는 System Call이다.

예시로, 기존에 존재하는 `getpid`시스템 콜을 통해 자신의 pid정보를 알 수 있다.

여기서, 단순히 본인 프로세스의 ID를 반환할 뿐인 getpid가 왜 System Call로 되어있는 이유는, 모든 프로세스의 정보는 Kernal이 관리하기 때문이다.

프로세스 자신은 자신이 어떤 프로세스인지 스스로 알 수 없다. 이는 자신을 exec 시킨 Kernal에서만 알 수 있다.

`getppid`를 구현하기 위해서는 우선 자신의 프로세스 정보를 알아야 한다.

커널에서 관리하는 자신의 프로세스 정보인 `proc` 구조체에는 부모 프로세스 정보가 포인터로 존재한다. ( `struct proc *parent;` )

![xv6 OS의 proc.h에 선언되어 있는 proc 구조체](attachment:ca99f2b1-fb5b-4c5f-a251-47ca53add515:image.png)

xv6 OS의 proc.h에 선언되어 있는 proc 구조체

`getppid` 호출 시 본인 프로세스의 `parent` 의 `pid` 멤버변수를  반환하도록 구현하면 된다.

## 4. System Call 호출 과정 정리

---

```c
//pidtest.c
#include "types.h"
#include "stat.h"
#include "user.h"

int main(int argc, char ** argv) {
	
	printf(1, "PID : %d\nPPID : %d\n", getpid(), getppid());
	
	exit();
}
```

`pidtest`는 `getpid()`와 `getppid()` System Call을 실행하는 Application이다.

`pidtest`를 실행하였을 때 내부적으로 어떤 순서로 시스템 콜이 실행되는지를 알아보자.

1. `pidtest` application에서 다음과 같이 `getpid()` 와 `getppid()` System Call을 호출한다.
2. `getpid()` 와 `getppid()` 를 포함한 System Call은 `user.h` 에 선언되어 있다.
3. 해당 함수의 본문은 `usys.S` 의 `SYSCALL(name)` 매크로를 통해 <unknown> getpid()와 같이 선언되어있으며, 이를 호출한다.
4. `getpid()` 프로시저가 호출되면, `%eax` 레지스터에 호출 번호를 저장 후 `$T_SYSCALL` 인터럽트를 발생시킨다.
5. `$T_SYSCALL` 인터럽트는 syscall() 함수를 실행하나, 해당 내용은 분석하지 못하였다.
6. `syscall()` 함수가 호출되면, `eax` 레지스터에 저장되어있는 호출 번호에 맞는 sys_XXXX() 함수를 호출한다. (getppid의 경우 22 : `syscall.h` 참고)
7. 결론적으로 `sys_getpid()` 와 `sys_getppid()` 가 실행된다.

> 변경해야 할 파일 : `user.h` , `usys.S` , `syscall.h` , `syscall.c` , `sysproc.c`

### [usys.S] : xv6의 System Call 프로시저 리스트

> .S는 전처리기를 거치는 어셈블리어 코드 파일이다. C언어의 전처리문을 사용할 수 있다.
> 

usys.S는 System Call Wrapper로, user.h에서 프로토타입 선언된 System Call 함수의 원형을 정의한다.

```nasm
#include "syscall.h"
#include "traps.h"

#define SYSCALL(name) \
  .globl name; \
  name: \
    movl $SYS_ ## name, %eax; \
    int $T_SYSCALL; \
    ret

SYSCALL(fork)
SYSCALL(exit)
#...(생략)...
SYSCALL(uptime)
SYSCALL(getppid) #getppid System Call 추가
```

SYSCALL 매크로를 사용하면 System Call을 정의하는 코드가 생성된다.

```nasm
#define SYSCALL(name) \
  .globl name; \ #name 으로 글로벌 심볼을 선언
  name: \ #name 심볼의 코드
	  # "SYS_($name)"은 syscall.h에 선언되어엤는 System Call 번호를 %eax 레지스터에 저장한다.
    movl $SYS_ ## name, %eax; \ 
    
    # $T_SYSCALL은 traps.h에 64로 define 되어있으며, 인터럽트 벡터 64번에 전송한다.
    int $T_SYSCALL; \
    
    # 프로시저가 종료되며 %eax 레지스터의 값을 리턴한다. 
    ret;
```

movl $SYS_ ## name, %eax;
eax 레지스터에 시스템 콜 번호를 저장해놓고

int $T_SYSCALL; 로 인터럽트를 걸면
eax 레지스터에 저장되있는 번호에 맞는 시스템 콜을 실행한다

> **결론**
SYSCALL( ) 매크로에 이름을 넣으면
그 이름으로 커널한테 시스템 콜 인터럽트 거는 프로시저가 컴파일 타임에 생긴다
예시 : SYSCALL(getpid) 실행 시 - 실제로는 컴바일된 바이너리가 생성

### [syscall.c] : 실제 System Call 함수 포인터 Wrapper

`usys.S` 에서 선언된 SYS_($name) 프로시저가 실행되면 `syscall()`을 실행한다.

`syscall()` 에는 SYSCALL() 프로시저 실행 시에 저장된 %eax 레지스터에 있는 시스템 콜 번호를

포인터 함수 배열인 `syscalls()` 의 index에 맞게 선언된 함수를 실행한다.

각 System Call 함수들은 extern sys_XXXX()로 선언하여 외부(`sysproc.c`)에 선언된 함수를 가져온다.

`syscalls()` 는 함수 포인터 배열로, 여러개의 함수가 배열로 들어가있다.

해당 배열에는 위에서 `sysproc.c` 에서 가져온 함수들이  `syscall.h` 에 선언된 번호에 맞게 원소로 배치되어 있다.

결론적으로는, 아래에 있는 순서대로 System Call이 실행된다.

<aside>
💡

**함수 실행 순서**
#define SYS_getppid 22 //syscalls 포인터 함수 배열의 index

1. syscall() 호출
2. syscalls[ SYS_getppid ]() 호출
3. sys_getppid() - `sysproc.c` //syscalls[SYS_getppid]() == sys_getppid()

</aside>

```c
extern int sys_getppid(void);

/* 
  syscalls[]() : System Call이 들어있는 포인터 함수 배열
*/
static int (*syscalls[])(void) = {
/* ... 생략 ... */
[SYS_link]    sys_link,
[SYS_mkdir]   sys_mkdir,
[SYS_close]   sys_close,
[SYS_getppid] sys_getppid,
};

/*
  syscall을 호출하면 호출한 프로세스의 
  eax 레지스터에 저장되어있는 번호에 맞는 System Call을 호출한다.
*/
void
syscall(void)
{
  int num;
  struct proc *curproc = myproc();

  num = curproc->tf->eax; //본인 proc 구조체에 있는 eax 레지스터를 가져옴
  if(num > 0 && num < NELEM(syscalls) && syscalls[num]) { //eax레지스터의 값에 이상이 없다면
    curproc->tf->eax = syscalls[num]();  //syscalls에 있는 System Call을 실행
  } else {
    cprintf("%d %s: unknown sys call %d\n",
            curproc->pid, curproc->name, num);
    curproc->tf->eax = -1;
  }
}
```

```c
{
```

## 6. getppid System Call 구현

---

```c
int
sys_getppid(void)
{
  return myproc()->parent->pid; //내 프로세스의 -> 부모 프로세스의 -> PID == PPID
}
```

`myproc()` 함수를 실행하면, 본인 프로세스의 `proc` 구조체가 반환된다.

위에서도 설명했다시피, `proc` 구조체에는 본인의 `pid` 와 부모 프로세스의 `proc` 구조체의 포인터가 있다.

부모 프로세스의 `proc` 으로 접근하여 `pid` 멤버 변수 값이 `ppid` (Parent Process ID) 가 된다.

해당 값을 반환한다.

## 추가분석 : Process 실행 시 System Call 호출 순서

모든 System Call은 `syscall()` 함수를 거쳐간다.

이를 이용하여 OS에서 System Call을 실행하는 순서를 출력해보았다.

`syscall()` 함수를 다음과 같이 수정하였다.

```c
const char* syscall_name(int num) {
    switch (num) {
        case 1:  return "SYS_fork";
        case 2:  return "SYS_exit";
        case 3:  return "SYS_wait";
        case 4:  return "SYS_pipe";
        case 5:  return "SYS_read";
        case 6:  return "SYS_kill";
        case 7:  return "SYS_exec";
        case 8:  return "SYS_fstat";
        case 9:  return "SYS_chdir";
        case 10: return "SYS_dup";
        case 11: return "SYS_getpid";
        case 12: return "SYS_sbrk";
        case 13: return "SYS_sleep";
        case 14: return "SYS_uptime";
        case 15: return "SYS_open";
        case 16: return "SYS_write";
        case 17: return "SYS_mknod";
        case 18: return "SYS_unlink";
        case 19: return "SYS_link";
        case 20: return "SYS_mkdir";
        case 21: return "SYS_close";
        case 22: return "SYS_getppid";
        default: return "Unknown syscall";
    }
}

void
syscall(void)
{
  int num;
  struct proc *curproc = myproc();

  num = curproc->tf->eax;
  if(num > 0 && num < NELEM(syscalls) && syscalls[num]) {
    cprintf("\nSyscall by [%s] - Call %s(%d) : ", curproc->name, syscall_name(num), num);
	
    curproc->tf->eax = syscalls[num]();
  } else {
    cprintf("%d %s: unknown sys call %d\n",
            curproc->pid, curproc->name, num);
    curproc->tf->eax = -1;
  }
}
```

호출되는 System Call 번호를 보기 쉽게 하기 위하여 `syscall_name()` 함수를 선언하였다.

호출 번호를 인자로 전달하면 System Call 이름을 char * 형식으로 반환한다.

위와 같이 구현한 후, xv6 OS를 실행시켜보았다.

```csharp

//init 프로세스가 OS 구동을 위한 initial을 시작한다.

Syscall by [init] - Call SYS_dup(10) :
Syscall by [init] - Call SYS_dup(10) :

Syscall by [init] - Call SYS_write(16) : n
```
