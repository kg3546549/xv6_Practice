# System Call

System Call은 **커널에 내장되어 있는 기본적인 인터페이스**이다.

* OS 컴파일 시 **커널과 함께 컴파일된다.**
* **사용자 프로그램(User Program)이 커널 기능을 사용할 수 있도록 하는 인터페이스**이다.
* 사용자 프로그램은 직접 커널을 조작할 수 없기 때문에 **System Call을 통해서만 커널 기능을 사용할 수 있다.**

즉,

```
User Program → System Call → Kernel
```

형태로 동작한다.

---

# 8대 System Call

대표적으로 운영체제에서 가장 기본이 되는 System Call은 다음 8가지이다.

## Process System Call

### 1. fork()

프로세스를 생성한다.

* 부모 프로세스를 **복사하여 자식 프로세스를 생성**
* 부모와 자식은 **거의 동일한 메모리 공간**을 가진다.
* 반환값이 서로 다르다.

| 반환값    | 의미      |
| ------ | ------- |
| 0      | 자식 프로세스 |
| 자식 PID | 부모 프로세스 |
| -1     | 실패      |

---

### 2. exec()

현재 프로세스의 메모리를 **새 프로그램으로 교체하여 실행**한다.

즉,

```
기존 프로그램 → 삭제
새 프로그램 → 실행
```

대표적인 함수

```
execl()
execv()
execle()
execvp()
```

---

### 3. wait()

부모 프로세스가 **자식 프로세스가 종료될 때까지 대기**한다.

```
wait(&status)
```

자식 프로세스 종료 시

* 종료 상태
* 종료 원인

등을 부모에게 전달한다.

---

### 4. exit()

프로세스를 종료한다.

실제로 커널에서 사용하는 것은

```
_exit()
```

이다.

일반적으로 사용하는 `exit()` 는 **C 라이브러리 함수**이며 내부적으로 `_exit()`를 호출한다.

---

# File I/O System Call

파일 입출력에 사용되는 System Call이다.

---

### 1. open()

파일을 연다.

```
fd = open("file.txt", O_RDONLY);
```

* 파일을 열면 **파일 디스크립터(File Descriptor)** 를 반환한다.

---

### 2. close()

파일을 닫는다.

```
close(fd);
```

---

### 3. read()

파일을 읽는다.

```
read(fd, buffer, size);
```

---

### 4. write()

파일을 쓴다.

```
write(fd, buffer, size);
```

---

# Shell에서 프로그램 실행 과정

shell (예 : bash)에서 프로세스를 실행하면 다음과 같이 동작한다.

예

```
$ ls
```

---

## 실행 순서

1️⃣ shell에서 `ls` 입력

2️⃣ `fork()` 실행
→ **자식 프로세스 생성**

3️⃣ `exec()` 실행
→ 자식 프로세스가 `ls 프로그램`으로 교체됨

4️⃣ 부모 프로세스(shell)가 `wait()` 실행
→ 자식 프로세스 종료 대기

5️⃣ `ls` 프로그램 실행
→ 현재 디렉토리 파일 목록 출력

6️⃣ 프로그램 종료 시 `exit()` 호출

7️⃣ 자식 프로세스 종료

8️⃣ 부모 프로세스(shell)가 종료 상태 반환받음

```
shell
  └ fork()
       └ child process
            └ exec(ls)
                 └ exit()
```

---

### wait()를 하지 않으면?

shell과 ls가 **동시에 실행**된다.

스케줄러가

```
shell → ls → shell → ls
```

이런 식으로 번갈아 실행하기 때문에

```
$ ls
$ file1
file2
file3
```

같이 출력 순서가 섞일 수 있다.

---

# ls 명령어

리눅스에서 `ls` 명령어는 **실행 파일(binary)** 이다.

보통 다음 위치에 존재한다.

```
/bin/ls
또는
/usr/bin/ls
```

파일 정보를 보면

```
-rwxr-xr-x 1 root root 142144 Sep 5 2019 ls
```

이것은 **파일 메타데이터(metadata)** 이다.

---

# stat() System Call

`ls` 명령어는 내부적으로 다음 System Call을 사용한다.

```
stat()
```

파일의 상태 정보를 가져온다.

반환값은 **stat 구조체**이다.

```c
struct stat {
    dev_t     st_dev;     
    ino_t     st_ino;     
    mode_t    st_mode;    
    nlink_t   st_nlink;   
    uid_t     st_uid;     
    gid_t     st_gid;     
    dev_t     st_rdev;    
    off_t     st_size;    
    blksize_t st_blksize; 
    blkcnt_t  st_blocks;  

    struct timespec st_atim;  
    struct timespec st_mtim;  
    struct timespec st_ctim;  
};
```

이 구조체는 **파일 메타데이터**를 포함한다.

---

# ls 출력 분석

예시

```
-rwxr-xr-x 1 root root 142144 Sep 5 2019 ls
```

각 항목을 분석해보면 다음과 같다.

---

# 1. 파일 종류 + 접근 권한

```
-rwxr-xr-x
```

## 파일 종류

맨 앞 1글자는 **파일 타입**이다.

| 문자 | 의미      |
| -- | ------- |
| -  | 일반 파일   |
| d  | 디렉터리    |
| l  | 심볼릭 링크  |
| b  | 블록 디바이스 |
| c  | 문자 디바이스 |
| p  | FIFO    |
| s  | 소켓      |

현재 파일

```
-
```

→ **일반 파일**

---

## 접근 권한

```
rwxr-xr-x
```

세 부분으로 나뉜다.

```
[Owner][Group][Others]
```

```
rwx  r-x  r-x
```

| 문자 | 의미      |
| -- | ------- |
| r  | read    |
| w  | write   |
| x  | execute |
| -  | 권한 없음   |

### Owner

```
rwx
```

읽기 + 쓰기 + 실행 가능

---

### Group

```
r-x
```

읽기 + 실행 가능

---

### Others

```
r-x
```

읽기 + 실행 가능

---

# 2. 링크 수

```
1
```

stat 구조체의

```
st_nlink
```

값이다.

**하드 링크 개수**를 의미한다.

---

## 하드 링크 생성

```
ln a.txt b.txt
```

이 경우

```
a.txt
b.txt
```

두 파일은 **같은 inode를 공유**한다.

---

```
root
```

stat 구조체의

```
st_uid
```

이다.

---

# 4. 그룹

```
root
```

stat 구조체의

```
st_gid
```

이다.

---

# 5. 파일 크기

```
142144
```

```

```

단위는 **Byte**이다.

---

```

```

```

```

이다.

# 7. 파일 이름


파일 이름이다.

# Sticky Bit

Sticky Bit는 **특수 권한 비트 중 하나**이다.

주로 **디렉터리에서 사용된다.**

대표적인 예

```
/tmp
```

권한

```
drwxrwxrwt
```

마지막 `t` 가 Sticky Bit이다.

---

## Sticky Bit 의미

디렉터리 안의 파일을

**파일 소유자만 삭제 가능**

하게 만든다.

예

```
/tmp
```

여러 사용자가 파일을 만들 수 있지만

다른 사용자의 파일을 삭제할 수 없다.

---

# SetUID

특수 권한 중 하나이다.

```
-rwsr-xr-x
```

`s` 가 보이면 **SetUID**

---

## 의미

프로그램을 실행하면

**파일의 소유자 권한으로 실행된다.**

대표적인 예

```
passwd
```

```
-rwsr-xr-x 1 root root ...
```

사용자가 passwd 실행

→ root 권한으로 실행

→ `/etc/shadow` 수정 가능

---

# Linux의 User ID

리눅스에는 3가지 User ID가 있다.

---

## 1. Real User ID (RUID)

실제 로그인한 사용자 ID

---

## 2. Effective User ID (EUID)

**권한 검사에 사용되는 ID**

SetUID 프로그램 실행 시

EUID가 **파일 소유자**가 된다.

---

## 3. Saved User ID

EUID를 저장하는 용도

권한 복구에 사용된다.

---

# 프로세스 상태

프로세스는 여러 상태를 가진다.

대표적인 상태는 다음과 같다.

---

## 1. Running

CPU에서 실행 중

---

## 2. Ready

CPU를 기다리는 상태

---

## 3. Waiting (Blocked)

I/O 작업을 기다리는 상태

예

* 디스크 읽기
* 네트워크 대기

---

## 4. Terminated

프로세스 종료

---

## 5. New

프로세스 생성 상태

---

# 인터럽트 종류

CPU가 현재 작업을 멈추고 **다른 작업을 처리하도록 하는 신호**

---

## Timer Interrupt

일정 시간마다 발생

→ **CPU 스케줄링에 사용**

---

## I/O Interrupt

I/O 작업 완료 시 발생

예

```
디스크 읽기 완료
네트워크 수신 완료
```

---

# 정리

User Program이 OS 기능을 사용하려면

```
System Call
```

을 사용해야 한다.

대표적인 흐름

```
shell
  └ fork()
      └ exec()
          └ wait()
              └ exit()
```

파일 정보 조회는

```
stat()
```

System Call을 통해 이루어진다.
